存在问题

如果一个页面同时存在两个富文本框，使用上述配置就会出现互相冲突的问题，看了源代码之后，具体原因不言而喻：

解决办法

先来看一段源代码：
$.fn.wysiwyg.defaults = {
    hotKeys: {
        'ctrl+b meta+b': 'bold',
        'ctrl+i meta+i': 'italic',
        'ctrl+u meta+u': 'underline',
        'ctrl+z meta+z': 'undo',
        'ctrl+y meta+y meta+shift+z': 'redo',
        'ctrl+l meta+l': 'justifyleft',
        'ctrl+r meta+r': 'justifyright',
        'ctrl+e meta+e': 'justifycenter',
        'ctrl+j meta+j': 'justifyfull',
        'shift+tab': 'outdent',
        'tab': 'indent'
    },
    toolbarSelector: '[data-role=editor-toolbar]',
    commandRole: 'edit',
    activeToolbarClass: 'btn-info',
    selectionMarker: 'edit-focus-marker',
    selectionColor: 'darkgrey',
    dragAndDropImages: true,
    fileUploadError: function (reason, detail) { console.log("File upload error", reason, detail); }
};
 看到这是默认配置，如果在初始化的时候直接$('#editor').wysiwyg();这样写，就是使用的默认配置，我们可以给两个富文本框起不同的
 toolbarSelector: '[data-role=editor-toolbar]',
 比如一个叫NameA:
 `<div class="btn-toolbar" data-role="NameA" data-target="#editor">`,
 另一个叫NameB:
 `<div class="btn-toolbar" data-role="NameB" data-target="#editor">`
 ，然后初始化富文本框A的时候这样写，
 $('#editor').wysiwyg({
  toolbarSelector:'[data-role=NameA]'
 });
 初始化富文本框B的时候这样写，
  $('#editor').wysiwyg({
  toolbarSelector:'[data-role=NameB]'
 });
就解决了冲突，是不是很简单~~~
